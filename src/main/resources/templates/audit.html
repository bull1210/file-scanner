Windows Security Event Log

Reads Security Event Log (Event IDs 4663, 4660, 4670) for deep auditing.

  • Detects: Read, Write, Delete, Permission changes
  • Shows: user, domain, process name
  • Requires Administrator + Object Access auditing enabled
  • Can query historical events up to 7 days back

To enable Windows file auditing, run these commands as Administrator:

# 1. Enable Object Access auditing
auditpol /set /subcategory:"File System" /success:enable /failure:enable

# 2. Set SACL on the folder (audit Everyone's read/write/delete)
icacls "" /audit:Everyone:(OI)(CI)(R,W,D,DC,WDAC,WO)

# 3. Restart this app as Administrator
